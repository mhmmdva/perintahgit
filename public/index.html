cd (nama_folder) : change directory , untuk melakukan perpindahan atau memasuki pada suatu folder yang di inginkan
git init : untuk menginisialisasi, dengan cara membuat repositori pada file local sehingga menjadi master
git status : untuk melihat status informasi pada git, apakah ada yang diperbaharui atau tidak
git add (nama_file.html): untuk memindahkan file yang akan di commit ke staging area
git add . : untuk memindahkan file yang akan di commit ke staging area secara langsung seperti ada penambahakan file baru atau perubahan codes
untracked files : ada penambahan file baru yang harus kita pindahkan ke dalam staging area
git log : melihat history pada commit
git log -3 : melihat history pada commit sebanyak 3x saja
git log -- index.html : melihat history perubahan secara spesifik pada file index.html saja
git log --all --decorate --oneline --graph : untuk melihat graph pada branch dengan commit hash dan ada commit massages
alias graph="git log --all --decorate --oneline --graph" untuk membuat suatu alias
git checkout 7767bsq -- index.html : mengembalikan file yang sudah kita hapus dengan memasukkan code commit pertama (7767bsq) sebannyak 7 digit dan file yang sudah dikembalikan akan masuk ke dalam staging area.
git checkout (nama_branch): untuk berpindah branch aktif bernama master ke branch yang lain
git checkout -b (nama_branch): membuat branch sekaligus untuk mengaktifkan branch tersebut
git clone (url_git_folder)" untuk mengclone folder dari github, dan jika download maka kita tidak terhubung ke remote
git remote ketika mau ngeremote, maka remote harus punya nama dan secara otomatis nama default dari remote adalah origin
git remote -v melihat data lebih detail tidak hanya nama remote
git remote add origin (url_git) untuk membuat remote baru karena di awal membuat folder pada dekstop bukan pada github dan pada bagian README.file itu jangan di checklist
git push -u origin master -u untuk mengeset upstream, jadi ketika kedepannya ingin push lagi tinggal mengetik git push saja
