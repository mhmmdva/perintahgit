Materi Git Merged!

    Fast Forward Merged

  • untuk melakukan merge pada branch, yaitu kita harus pindah terlebih dahulu ke dalam branch master setelah pindah ke dalam branch master maka kita bisa melakukan git merge (nama_branch), ini untuk melakukan fast forward

  • dan untuk menghapus branch nya karena sudah di merge sebelumnya, kita bisa menggunakan perintah

  • git branch --merged : yaitu git akan secara otomatis menghapus branch yang sudah di merge tetapi posisi kita harus berada di dalam branch master

  • git branch -d (nama_branch) : untuk menghapus branch setelah di merged

  • git branch -D (nama_branch) : menghapus branch secara paksa tanpa adanya merge dengan alasan file yang dibuat itu tidak terpakai

Three-way Merged

  • ini adalah sebuah branch yang akan terjadi ketika, branch satu dengan branch yang lain itu tidak memiliki hubungan satu sama lain

  • git branch --merged : yaitu git akan secara otomatis menghapus branch yang sudah di merge tetapi posisi kita harus berada di dalam branch master

  • git branch -d (nama_branch) : untuk menghapus branch setelah di merged

Conflict Merged

  • jika pada satu file ada yang mengubah atau mengganti isi dari codes tersebut yang menyebabkan error/conflict pada saat di commit, itu mengharus kan kita sebagai developer untuk memilih mana yang akan kita gunakan.

    HEAD CURRENT : jika ada hal seperti ini, berarti ini menandakan file yang aktif yang sedang di modifikasi nama_branch (incoming change) : jika ada hal seperti ini, berarti ini adalah file yang sebelumnya sudah di modifikasi terlebih dahulu

    • Resolve conflict Merged

  • dengan cara memilih codes atau baris mana yang ingin kita pakai dan kita hapus, jika sudah maka save. Dan masuk ke dalam git status untuk melihat status nya
